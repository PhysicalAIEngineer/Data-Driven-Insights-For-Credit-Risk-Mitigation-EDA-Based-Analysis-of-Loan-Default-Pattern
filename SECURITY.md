# 🔐 Security Policy

## 📌 Supported Versions

The following versions of this project are currently supported with security updates:

| Version              | Supported |
| -------------------- | --------- |
| Latest (main branch) | ✅ Yes     |
| Older versions       | ❌ No      |

We recommend always using the **latest version** of the project to ensure maximum security.

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### 📩 How to Report

* Email: **[[your-email@example.com](mailto:your-email@example.com)]**
* Subject: `Security Vulnerability Report - Credit Risk EDA Project`

---

### 📝 What to Include

Please provide as much detail as possible:

* Description of the vulnerability
* Steps to reproduce the issue
* Potential impact
* Screenshots or proof-of-concept (if available)

---

## 🔒 Responsible Disclosure

* Do **NOT** publicly disclose the vulnerability before it is addressed
* Allow maintainers reasonable time to investigate and fix the issue
* Avoid exploiting the vulnerability beyond proof-of-concept

---

## ⚡ Response Process

Once a report is received:

1. 📬 Acknowledgement within **48 hours**
2. 🔍 Investigation and validation
3. 🛠️ Fix development
4. 📢 Public disclosure (if applicable)

---

## 🛡️ Security Best Practices

To ensure safe usage of this project:

* 🔑 Do not expose sensitive data (API keys, credentials)
* 📁 Use trusted datasets only
* 🔄 Keep dependencies updated
* 🧪 Validate inputs and outputs carefully

---

## ⚠️ Scope

This project is primarily focused on **data analysis and research**.
Security concerns may include:
